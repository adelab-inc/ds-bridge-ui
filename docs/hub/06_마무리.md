# 마지막 장. 사람들이 이 플랫폼을 기억하는 방식

---

## 이 플랫폼을 오래 쓰게 된 사람들은

어느 순간 이렇게 말하기 시작한다:

> "디자인 시스템은 거기서 돌려보면 돼."
> "스토리북? 그거 그냥 저기 올려."
> "문서 보지 말고, 여기서 직접 만져봐."

이때 이 플랫폼은 **툴이 아니라 습관**이 된다.

---

## 이 플랫폼이 해결한 문제는 단순하지 않다

겉으로 보면 이렇게 보일 수 있다:
- 디자인 시스템을 모아놓은 사이트
- 스토리북을 좀 더 편하게 보는 도구
- AI로 UI를 만들어주는 서비스

**하지만 실제로 바꾼 건 이것이다:**

> **디자이너와 개발자가 '같은 걸 보면서 이야기할 수 있는 방법'**

---

## 예전의 대화

- "이 버튼은 primary로 써야 해요."
- "문서에 그렇게 써 있잖아요."
- "아니, 실제로 보면 다를 수도 있어요."

말이 많고, 확인이 느렸다.

---

## 이 플랫폼 이후의 대화

- "이 버튼, 여기서 이렇게 바꾸면 이 느낌이에요."
- "아, 그러면 이 페이지에서는 secondary가 맞겠네요."
- "그럼 이 조합으로 가죠."

**말 대신 실행이 대화가 된다.**

---

## 이 플랫폼의 방향성

우리는 더 많은 기능을 만들려고 하지 않는다.
우리는 더 똑똑한 AI를 자랑하려 하지 않는다.

**우리가 지키는 방향은 단 하나다:**

> **"디자인 시스템을 설명의 대상이 아니라 실행의 기준으로 만든다."**

---

## 이 플랫폼이 되고 싶은 위치

| 아님 | 맞음 |
|------|------|
| 디자인 툴 ❌ | |
| 코드 생성기 ❌ | |
| 협업 툴 ❌ | |
| | **디자인 시스템을 '운영'하는 기반** ⭕ |

디자인 시스템이 생길 때마다 사람들이 자연스럽게 떠올리는 곳.

> "아, 이건 거기서 먼저 돌려보자."

---

## 사람들이 좋아하는 이유 (아주 단순하게)

사람들이 이 플랫폼을 좋아하게 되는 이유는 거창하지 않다:

- 설정이 거의 없다
- 설명을 읽지 않아도 된다
- 결과를 바로 볼 수 있다
- 틀려도 안전하다

> **"결정하기 전에, 먼저 실행해볼 수 있어서."**

## 마지막으로, 이 프로젝트의 정체성

이 프로젝트는 빠르게 성장하는 서비스가 되지 않을 수도 있다.

하지만 한 번 자리 잡으면 **바꾸기 어려운 기반**이 된다.

왜냐하면 우리는 **사람들의 결정 방식을 바꾸기 때문이다.**

```
설명 → 실행
추측 → 확인
문서 → 경험
```
