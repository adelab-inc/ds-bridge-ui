# Design System Runtime Hub - 요약

> **"설명 말고, 실행하세요."**

---

## 핵심 컨셉

디자인 시스템 문서 사이트가 아닌, **디자인 시스템을 직접 실행해보는 플랫폼**

| 구성 요소 | 역할 |
|-----------|------|
| **JSON** | 컴포넌트 정의 (props, 구조) |
| **Storybook** | 정의를 실제로 그려주는 실행기 |
| **플랫폼** | JSON + Storybook을 연결하여 실행 가능한 환경 제공 |

---

## 5막 로드맵

### 1막: Authority 확보 (Day 1-3)

**목표**: "이 서비스는 진짜다"라는 인상을 10초 안에 전달

**방법**:
- MUI, Ant Design, Carbon 등 검증된 디자인 시스템을 즉시 실행
- 복잡한 설정/설명 없이 `[Run MUI]` 버튼 하나로 시작
- 편집 기능 없음 - 오직 **증명**만

**성공 기준**:
- [ ] 유명 DS가 즉시 실행됨
- [ ] 컴포넌트 클릭 시 실제 화면 표시
- [ ] 설명 없이도 서비스 이해 가능

---

### 2막: 편집 도입 (Day 4-6)

**목표**: 보기만 하던 DS를 직접 만져볼 수 있게

**핵심 원칙**:
- 컴포넌트 **정의**(JSON)는 수정하지 않음
- 개별 **인스턴스의 실행 상태**(props)만 변경

**UI 구조**:
```
┌─────────────────────┬─────────────────────┐
│  구성 트리 (왼쪽)    │  실행 화면 (오른쪽)  │
│  ▸ Button#1         │  [실제 컴포넌트]     │
│  ▸ Button#2         │                     │
└─────────────────────┴─────────────────────┘
```

**편집 흐름**:
1. 구성 트리에서 인스턴스 선택
2. Props 패널에서 값 변경 (variant, size 등)
3. 선택한 인스턴스만 즉시 반영

**성공 기준**:
- [ ] 구성 트리에서 컴포넌트 선택 가능
- [ ] props 변경 시 화면 즉시 반영
- [ ] 선택한 인스턴스만 변경됨
- [ ] JSON/코드 변경 없음

---

### 3막: AI 조합 (Day 7-10)

**목표**: AI가 '생성'이 아닌 '조합'을 수행

**AI가 하지 않는 것**:
- 새로운 UI를 상상해서 그리기
- DS에 없는 컴포넌트 만들기
- props를 임의로 만들기

**AI가 하는 것**:
- DS 안에 있는 컴포넌트 중 목적에 맞는 것을 골라 페이지 구조 생성

**흐름**:
```
사용자 입력: "로그인 페이지 만들어줘"
     ↓
AI가 JSON 분석 (관련 컴포넌트만 슬라이스)
     ↓
조합 데이터 출력:
  Login Page
   ├─ TextField (Email)
   ├─ TextField (Password)
   └─ Button (Primary)
     ↓
Storybook에서 즉시 실행
```

**핵심**: `AI → 기본 조합` → `사람 → 미세 조정`

**성공 기준**:
- [ ] 한 줄 프롬프트로 페이지 구성 생성
- [ ] AI가 고른 컴포넌트가 실제 DS에 존재
- [ ] 결과 화면 즉시 실행
- [ ] 2막 방식으로 추가 편집 가능

### 4막: 사용자 DS 연동 (Month 1)

**목표**: "우리 회사 DS로도 된다"

**방법**:
```bash
npx ds-hub extract
```
- Storybook 분석 → JSON 추출 → ds.json 생성
- 업로드 후 Private DS 페이지 생성

**변화**:
| 이전 | 이후 |
|------|------|
| "스토리북 주소 공유할게요" | "이 링크 보세요" |
| 문서로 설명 | 직접 만져보기 |

**성공 기준**:
- [ ] 자체 Storybook에서 JSON 추출 가능
- [ ] 업로드 후 즉시 Private DS 페이지 생성
- [ ] 1-3막 기능 동일하게 작동
- [ ] 팀원에게 링크 공유 시 재현됨

---

### 5막: Workshop (Month 2-3)

**목표**: DS를 확장 가능한 실행 단위로 만들기

**Workshop이란**: 기존 DS 위에 덧붙일 수 있는 확장들의 공간

**예시 확장**:
- Dashboard Charts Pack
- Admin Table Pack
- Dark Mode Patch
- Mobile Layout Set

**핵심 원칙**:
- 기존 DS 수정 없이 기능 확장
- 포크/파편화 방지
- AI가 원본 + 확장을 함께 고려

**성공 기준**:
- [ ] 하나의 DS에 여러 확장 적용 가능
- [ ] 기존 시스템 수정 없이 기능 확장
- [ ] AI가 원본 + 확장 함께 조합
- [ ] DS 포크 없음
