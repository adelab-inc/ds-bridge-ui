# 5막. Workshop이 열린다

---

## 이 단계에서 질문이 다시 바뀐다

4막까지 온 사용자는 이미 이렇게 생각한다:
- "우리 팀 디자인 시스템을 여기서 돌릴 수 있다"
- "props도 만져보고, AI로 조합도 된다"
- "이건 충분히 실무에 쓸 수 있다"

그래서 다음 질문이 나온다:

> **"근데… 이 디자인 시스템에 내가 필요한 게 하나 빠져 있으면 어떡하지?"**

5막은 이 질문에 답하는 단계다.

---

## 5막의 핵심 변화

**이전까지** 디자인 시스템은 보통 이랬다:
- 한 번 만들면 고정
- 부족하면 직접 수정
- 수정하면 포크 → 파편화

**5막부터는** 이렇게 바뀐다:

> **디자인 시스템은 '완성품'이 아니라 '확장 가능한 실행 단위'가 된다.**

---

## Workshop이란 무엇인가

Workshop은 새로운 툴이 아니다. 새로운 UI도 아니다.

**"이 디자인 시스템 위에 덧붙일 수 있는 것들의 공간"**이다.

예를 들면 이런 것들:
- 이 DS용 차트 팩
- 이 DS용 어드민 테이블 패턴
- 이 DS용 다크모드 패치
- 이 DS용 모바일 레이아웃 세트

**중요한 점:**
이것들은 디자인 시스템 전체를 바꾸지 않는다.
**기존 시스템 위에 얹힌다.**

---

## 실제 화면에서 벌어지는 일

사용자가 어떤 디자인 시스템을 보고 있다:

```
Acme Design System
```

이 시스템에는 기본 UI 컴포넌트만 있다. 차트가 없다.

사용자는 **Workshop 탭**을 연다:

```
Available Add-ons for Acme DS

- Dashboard Charts Pack
- Admin Table Pack
- Dark Mode Patch
```

사용자는 하나를 선택한다:

```
Install Dashboard Charts Pack
```

---

## 설치 이후 무슨 일이 일어나나

- 기존 디자인 시스템은 그대로
- 새로운 컴포넌트들이 추가된 것처럼 보인다

```
Components
- Button
- TextField
- Checkbox
- Chart (from Dashboard Pack)
```

이제 AI에게 이렇게 말할 수 있다:

> "이 디자인 시스템으로 차트 포함된 대시보드 페이지 만들어줘"

AI는 **원본 DS + 설치된 팩**을 같이 고려해서 조합한다.

---

## 여기서 중요한 점 하나

Workshop은 코드 마켓이 아니다.
npm처럼 설치하는 것도 아니다.

**Workshop은 '실행 가능한 조합의 공유 공간'이다.**

- 누군가 만든 패턴
- 누군가 정리한 조합
- 누군가 다듬은 확장

이 모든 게 **'실행 기준'으로 재사용**된다.

---

## 이 단계가 해결하는 오래된 문제

기존 디자인 시스템의 문제는 이거였다:
- "이건 우리한테 90%는 맞는데…"
- "나머지 10% 때문에 다시 만들어야 하네"

**5막 이후에는 이렇게 바뀐다:**

> "이 DS는 좋은데, 여기 이 팩 하나만 얹으면 되겠네."

**이게 파편화를 막는 구조다.**

---

## 누가 이걸 좋아하게 될까

### 디자인 시스템 만든 팀
- 자기 시스템이 확장되는 걸 본다
- 다른 사람들이 어떻게 쓰는지 알게 된다

### 사용하는 팀
- 새로 만들 필요 없이
- 필요한 것만 얹어서 쓴다

### 조직 전체
- 디자인 시스템이 "관리 대상"이 아니라
- **"살아있는 자산"**이 된다

---

## 5막에서 중요한 경계선

5막에서도 우리는 여전히 이것을 **하지 않는다**:
- ❌ 커뮤니티 기능을 먼저 만들지 않는다
- ❌ 댓글, 좋아요, 랭킹을 만들지 않는다
- ❌ 소셜 네트워크를 만들지 않는다

**Workshop은 조용히 열려 있다.**
필요한 사람만 와서 필요한 걸 가져간다.

---

## 성공 기준 (Definition of Done)

- [ ] 하나의 디자인 시스템에 여러 "확장"이 붙을 수 있다
- [ ] 기존 시스템을 수정하지 않고도 기능이 늘어난다
- [ ] AI가 원본 + 확장을 함께 고려해 조합한다
- [ ] 디자인 시스템이 포크되지 않는다

---

## 5막의 역할

5막은 규모를 키우는 단계가 아니다.
**5막은 질서를 만드는 단계다.**

> "디자인 시스템은 버리고 새로 만드는 게 아니라 덧붙여서 진화한다."

이 질서가 만들어지면, 이 플랫폼은 더 이상 선택지가 아니다.

---

## 다음으로

> "그래서 이 플랫폼이 사람들에게 어떤 방향으로 기억되길 원하는가?"

그 질문에 답하는 게 **마지막 장**이다.
