# 3막. AI가 '생성'이 아니라 '조합'을 한다 

---

## 이 단계의 전제

3막은 갑자기 AI가 똑똑해지는 단계가 아니다.
**3막은 사람이 이미 이해한 걸 AI가 대신 해주는 단계다.**

2막까지 사용자는 이미 이것을 손으로 느꼈다:
- 이 디자인 시스템에 어떤 컴포넌트가 있는지
- 어떤 props가 가능한지
- 어떤 조합이 자연스러운지
- 어떤 조합은 안 되는지

그래서 이제 질문이 바뀐다:

> "이걸 내가 매번 직접 하지 말고, AI가 대신 해줄 수는 없을까?"

---

## 먼저, AI가 하지 않는 것

3막에서 AI는 다음을 **하지 않는다**:
- ❌ 새로운 UI를 상상해서 그리지 않는다
- ❌ 디자인 시스템에 없는 컴포넌트를 만들지 않는다
- ❌ props를 마음대로 만들어내지 않는다

**즉, AI는 '창작자'가 아니다. AI는 '조합자'다.**

---

## 그럼 AI는 정확히 무슨 일을 할까?

AI가 하는 일은 딱 이거다:

> **"이 디자인 시스템 안에 이미 있는 것들 중에서
> 지금 목적에 맞는 인스턴스들을 골라서
> 페이지 구조를 만들어준다."**

---

## 실제 화면에서 벌어지는 흐름

### 1️⃣ 사용자가 한 줄을 입력한다

화면 상단에 입력창이 있다:

```
What do you want to build?

> 로그인 페이지 만들어줘
```

### 2️⃣ AI는 JSON을 본다 (전체가 아니라 일부만)

AI는 다음을 본다:
- 이 DS에 어떤 컴포넌트들이 있는지
- 각 컴포넌트에 어떤 props가 있는지
- 어떤 story들이 대표적인지

**중요한 점:**
AI는 전체 JSON을 다 읽지 않는다.
"로그인"과 관련된 컴포넌트만 슬라이스해서 본다.

### 3️⃣ AI의 결과는 '화면'이 아니라 '조합'이다

AI가 내놓는 결과는 JSX가 아니다. 이미지도 아니다.
**이런 조합 데이터다:**

```
Login Page
 ├─ TextField (Email)
 ├─ TextField (Password)
 └─ Button (Primary)
```

각 항목은 이렇게 명확하다:
- 어떤 컴포넌트인지
- 어떤 story를 쓰는지
- 어떤 props 상태인지

### 4️⃣ 이 조합이 바로 실행된다

이 조합은 곧바로 화면에 올라간다:

```
┌───────────────────────────┐
│ [ Email Input ]           │
│ [ Password Input ]        │
│ [  Login Button  ]        │
└───────────────────────────┘
```

**여기서 중요한 점:**
- 이 화면은 AI가 그린 그림이 아니다
- 스토리북에 이미 존재하던 컴포넌트 조합이다

---

## 그래서 3막의 감각은 이렇다

사용자는 이렇게 느낀다:

> "AI가 새 걸 만들어준 게 아니라 우리 시스템에서 맞는 걸 골라줬네"
> "그러면 이 결과는 믿을 수 있겠다"

**이게 '생성'과 '조합'의 차이다.**

---

## 3막에서 편집은 어떻게 이어지나?

여기서 2막과 자연스럽게 이어진다.

AI가 만든 화면은 끝이 아니다. **시작점이다.**

사용자는 다시 할 수 있다:
- 구성 트리에서 Button 선택
- variant 바꾸기
- size 바꾸기
- loading 켜보기

**즉:**
```
AI → 기본 조합
사람 → 미세 조정
```

이 흐름이 된다.

---

## 왜 이 방식이 중요한가

### 이 방식이 아니면 생기는 문제:
- AI 결과를 믿을 수 없다
- "왜 이 버튼이 여기 있지?" 설명이 안 된다
- 회사에 들고 가기 불안하다

### 이 방식이면:
- 모든 결과가 DS 안에서 나온다
- 결과를 설명할 수 있다
- 다시 실행할 수 있다
- 링크로 공유해도 그대로 재현된다

---

## 성공 기준 (Definition of Done)

- [ ] 한 줄 프롬프트로 페이지 구성이 나온다
- [ ] AI가 고른 컴포넌트가 실제 DS에 존재한다
- [ ] 결과 화면이 즉시 실행된다
- [ ] 사용자가 2막 방식으로 다시 편집할 수 있다

---

## 3막의 역할

3막은 AI를 자랑하는 단계가 아니다.
**3막은 사람의 결정을 가속하는 단계다.**

> "내가 할 수 있는 걸 AI가 먼저 해주네."

이 감각이 만들어지면, 이 플랫폼은 더 이상 실험 도구가 아니다.

---

## 다음으로

> "이거, 우리 회사 디자인 시스템으로도 되는 거야?"

그 질문에 답하는 게 **4막**이다.
