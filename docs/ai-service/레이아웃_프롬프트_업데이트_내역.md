## 2차: DataGrid 프롬프트 보강 (`b885c4d`)

### cellRenderer 규칙 변경

| 항목 | Before | After |
|------|--------|-------|
| 버튼 렌더링 | `ButtonCellRenderer` 사용, inline 함수 금지 | **Button 컴포넌트를 화살표 함수 cellRenderer로 직접 사용**, `ButtonCellRenderer` 사용 금지 |
| 금지 사유 | — | `ButtonCellRenderer`는 디자인 시스템 미적용, 파란색(`#007bff`) 하드코딩, variant/size 커스터마이징 불가 |

**변경 전:**
```tsx
// ❌ FATAL — inline cellRenderer KILLS the grid
cellRenderer: ButtonCellRenderer,
cellRendererParams: { onClick: (data) => { ... } }
```

**변경 후:**
```tsx
// ✅ Button 컴포넌트를 cellRenderer 화살표 함수로 직접 사용
cellRenderer: (params: any) => (
  <Button variant="outline" size="sm" onClick={() => handleDetail(params.data)}>
    상세
  </Button>
)
```

### 신규 추가 항목

| 항목 | 내용 |
|------|------|
| Checkbox Column Pattern | `onCheckboxChange`에서 반드시 `setRowData`로 rowData 상태 업데이트 필수. 안 하면 체크 즉시 해제됨 |
| rowData useState 규칙 | `const rowData = [...]` 금지 → `const [rowData] = useState([...])` 필수. 리렌더 시 새 배열 생성되어 체크박스 선택 해제, 스크롤 초기화 등 발생 |
| Checkbox Selection Pattern | `rowSelection="multiple"` 사용 시 `suppressRowClickSelection={true}` 필수. 없으면 행 아무 곳 클릭해도 체크박스 토글됨 |
| Event Handlers 가이드 | `onCellClicked`, `onRowSelected`, `onSelectionChanged`, `onCellValueChanged`, `onGridReady` 사용법 추가 |

### columnDefs 규칙 변경

| 규칙 번호 | Before | After |
|-----------|--------|-------|
| 2. cellRenderer | "ONLY use named components", inline 함수 KILLS GRID | "화살표 함수 또는 named component 사용", `ButtonCellRenderer` 사용 금지 |
| 4. rowData (신규) | — | 반드시 `useState` 또는 `useMemo`로 관리 |

### import 가이드 변경

```tsx
// Before
import { DataGrid, ButtonCellRenderer, CheckboxCellRenderer, ImageCellRenderer } from '@aplus/ui';

// After
import { DataGrid, CheckboxCellRenderer, ImageCellRenderer } from '@aplus/ui';
```
